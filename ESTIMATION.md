# 持仓参考估算

## 归属边界

- `functions/api/holding-estimates.js`：数据/runtime 层。代理并缓存公开持仓和行情，只允许项目指定的基金代码。
- `fund-estimator.js`：算法/数据策略层。计算持仓参考幅度、管理本地校准状态、结算正式净值误差。
- `index.html`：呈现/render 与语言/expression 层。官方估值和持仓参考使用不同标签。

## 计算口径

原始参考涨跌幅：

```text
raw = sum(持仓占基金净资产比例 * 股票当日涨跌幅) / 100
```

持仓比例直接使用基金季报的净资产占比，不将前十大持仓重新归一化到 100%。未披露仓位、现金、衍生品和无有效行情的证券按 0% 贡献处理，并通过“持仓覆盖”提示可能的低估幅度。

校准结果：

```text
estimate = clamp(beta * raw + bias, -10%, +10%)
```

初始 `beta = 1`、`bias = 0`。正式净值日期推进到已保存的预测日后，以前一正式净值计算实际日涨跌，记录方向命中和误差，并对 `beta`、`bias`、近期绝对误差做限幅更新。历史只保存在当前浏览器 `localStorage`，不上传份额或预测记录。

## 启用与降级

- 官方 `GSZ` 与 `GSZZL` 同时存在时始终使用官方盘中估值。
- 仅对项目允许清单内、缺少官方盘中估值的基金请求持仓参考。
- 有效持仓行情覆盖低于基金净资产的 20% 时不输出参考值。
- `007099` 是债券基金，股票持仓无法代表净值波动，保留正式净值。
- `012920` 涉及海外交易时段和汇率，在未建立隔夜行情及汇率模型前不输出参考值。
- 行情日期不是上海当日、持仓或行情请求失败时，保留正式净值并显示降级状态。

## 回归检查

1. 官方盘中估值存在时，页面不得被持仓参考覆盖。
2. 持仓参考必须显示“参考”、覆盖率、信心等级和校验状态。
3. 前十大持仓总占比不足 100% 时，计算不得归一化放大。
4. 同一正式净值日期只能结算一次误差。
5. 单只基金持仓请求失败不得拖垮其他基金。
